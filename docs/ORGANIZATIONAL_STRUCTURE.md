# Organizational Structure

## Core Team Roles & Responsibilities

### 1. Creative Director / Founder
**Primary Focus:** Strategy, Quality Control, Client Relations

**Responsibilities:**
- Final design and content approvals
- Client relationship management
- Strategic direction and vision
- Quality assurance standards
- Brand consistency oversight
- Business development

**Decision Authority:** Full
**Reports To:** Self (Founder)
**Team:** Oversees all departments

**Key Metrics:**
- Client satisfaction (NPS)
- Project quality score
- Timeline adherence
- Revenue targets

---

### 2. Lead Developer
**Primary Focus:** Technical Excellence, Code Quality, Architecture

**Responsibilities:**
- Architecture decisions and technical strategy
- AI-generated code review and validation
- Performance optimization
- Code quality standards
- Technology stack recommendations
- Technical debt management
- Developer tools & workflow optimization

**Decision Authority:** Technical decisions, code standards
**Reports To:** Creative Director
**Team:** Oversees development workflows

**Key Metrics:**
- Code quality score
- Performance metrics
- Deployment success rate
- Technical debt reduction

---

### 3. Project & Operations Manager
**Primary Focus:** Workflow Efficiency, Timelines, Automation

**Responsibilities:**
- Project timeline tracking
- Workflow automation setup and monitoring
- Client onboarding and communication
- Team coordination
- Deployment and release management
- Process optimization
- Documentation maintenance
- Tools and systems management

**Decision Authority:** Operational decisions, process changes
**Reports To:** Creative Director
**Team:** Coordinates all teams

**Key Metrics:**
- On-time delivery rate
- Automation coverage
- Client communication score
- Team efficiency metrics

---

## Team Communication Structure

### Daily Standup
- **Frequency:** Daily at 9:00 AM
- **Duration:** 15 minutes
- **Format:** Each role reports progress, blockers, upcoming tasks
- **Owner:** Project Manager

### Weekly Planning
- **Frequency:** Every Monday at 10:00 AM
- **Duration:** 30 minutes
- **Format:** Review projects, plan week, discuss priorities
- **Owner:** Creative Director

### Bi-weekly Review
- **Frequency:** Every other Friday at 4:00 PM
- **Duration:** 45 minutes
- **Format:** Review deliverables, client feedback, process improvements
- **Owner:** Creative Director

### Monthly Strategy Session
- **Frequency:** First Friday of month at 2:00 PM
- **Duration:** 60 minutes
- **Format:** Review metrics, plan improvements, discuss growth
- **Owner:** Creative Director

---

## Decision-Making Framework

### Strategic Decisions (Creative Director Authority)
- Company direction and vision
- Major client acquisition/loss
- Pricing changes
- New service offerings
- Hiring decisions
- Technology stack changes

### Tactical Decisions (Lead Developer + Project Manager)
- Technology selection (within approved stack)
- Process improvements
- Tool implementations
- Workflow optimization
- Timeline adjustments

### Operational Decisions (Project Manager Authority)
- Day-to-day scheduling
- Client communication timing
- Internal process adjustments
- Resource allocation
- Documentation updates

---

## Escalation Path

```
Issue/Decision
    ↓
Responsible Role (tries to solve)
    ↓
If blocked → Escalate to Manager Role
    ↓
If still blocked → Escalate to Creative Director
    ↓
If business-critical → Founder approval
```

## Performance Review Cycle

### Monthly Check-ins (Informal)
- One-on-one with supervisor
- 15-20 minutes
- Progress discussion
- Feedback exchange

### Quarterly Reviews (Formal)
- Comprehensive performance review
- Goal progress assessment
- Compensation discussion (if applicable)
- Development planning

### Annual Reviews (Strategic)
- Year-end evaluation
- Strategic alignment
- Career development
- Goals for next year

---

## Communication Protocols

### Async Communication (Preferred)
- Slack for immediate updates
- Notion for documentation
- GitHub for code discussions
- Email for formal records

### Sync Communication
- Daily standup (15 min)
- Weekly planning (30 min)
- Client calls (as scheduled)
- Emergency-only unscheduled meetings

### Client Communication
- Project Manager is primary contact
- Weekly status updates
- Bi-weekly progress reviews
- Creative Director for escalations/strategy

---

## Team Principles

1. **Transparency** — All decisions and progress documented and visible
2. **Autonomy** — Each role has clear authority over their domain
3. **Accountability** — Clear metrics and review cycles
4. **Collaboration** — Regular sync points and async coordination
5. **Continuous Improvement** — Monthly retrospectives and process adjustments
6. **Quality First** — All work meets established standards before delivery
7. **Client Focus** — Every decision considers client value and satisfaction
